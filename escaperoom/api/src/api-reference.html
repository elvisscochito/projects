Api Reference

  • Player

    A player represents a user account, one can create users, delete users, get user info, etc.

    • GET /player

      Returns list of ALL players

    • GET /player/kenny

      Returns ALL info related to a player, such as match history, total score, achivements, etc.

    • POST /player
      { "username": "kenny", "password": "password", "email": "kenny@mail.com" }

      Creates a new player

    • DELETE /player/kenny

      Deletes player of given username

    • POST /player/login
      { "username": "kenny", "password": "password" }

      Authenticates as player and returns token

  • Event

    An event represents some possible in-game action, such as getting an achivement, opening a door, etc. Events are associated with a NAME and a (optional) VALUE.

    • GET /event

      Gets all registered events

    • POST /event
      { "name": "OPEN_DOOR", "value": 1 }

      Creates a value with a name and its value

    • DELETE /event/:name

      Deletes the event of given name

  • Match

    A match represents a full game cycle, all other information is associated to exactly one match, such as "player x did y in match z".

    • GET /match

      Get list of all existing matches

    • GET /match/:id

      Get all information related to a match, such as

      • Match players
      • Duration
      • Game events
      • Total scores
      • Etc.

    • POST /match
      { "duration": 3456, "players": [ "kenny", "gabdiel", "emmanuel" ], "events": [ { "player": "kenny", "event": "OPEN_DOOR" } ] }

      Registers a new match, with all associated information

    • DELETE /match/:id

      Deletes the match of given id
